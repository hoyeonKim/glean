NAME

gln - search interface for glean

SYNOPSIS

gln [-h] [-d <db_dir>] [-v] [-n] [-N] [-C <context_lines>] [-e <errors>] [-s] [-g] [-D] <QUERY>

DESCRIPTION

gln searches a filesystem using an index previously generated by gln_index. The index specifies which files to search based on the query provided, and then a pipeline of one or more grep commands is assembled and executed.

Options

-h print help summary and exit.

-d <db_dir> set location of the .gln index directory (default: ~).

-v increase verbosity (can be used multiple times).

-n only print matching filenames, not content.

-N omit filenames from output.

-C <context_lines> set lines of context for NEAR searchers (default: 4).

-e <errors> allow up to <errors> errors when searching for matching tokens. Use this to search for a word, including misspellings. Requires nrgrep. The -e option's argument is passed verbatim to nrgrep.

-s do not implicitly wrap search token regexes in "^(token)$". This searches for all tokens that contain the search tokens as substrings.

-t print tokens matched by the search pattern and exit.

-g do not execute the grep pipeline, just print it and exit.

-D dump info about index database and exit. With -v option, also dump contents.

Queries

A query consists of one or more tokens and optional keywords. Each token represents a regular expression to search for, and keywords affect the interaction of multiple tokens. (Keywords must be in uppercase.)

a AND b Require both tokens to occur in the same line.

a OR b Either 'a' or 'b' must occur in the file; show lines containing either.

a NOT b Do not show results for files containing 'a' if they contain 'b' (on any line).

a NEAR b Search files that have 'a' within <context_lines> lines of 'b'. (Not yet implemented.)

Example

$ gln foo

Search all indexed files for "foo".

$ gln -d. foo bar

Search files for "foo" and "bar" that have an index in the current directory.

$ gln -d ~/dev/project/ stopword OR index AND NOT btree

Search all files indexed in ~/dev/project/ for lines containing either "stopword" or "index" that do not contain "btree".

$ gln "erl.*"

Search all files for all tokens matching the regular expression "erl.*" (e.g. "erlang").

EXIT STATUS

gln returns 0 on success or 1 on error.
